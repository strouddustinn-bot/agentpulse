# Join the paid beta

AgentPulse is onboarding a small group of founders and small teams who run Linux servers and want monitoring that can safely remediate repeat incidents.

[Email us to reserve a slot](mailto:support@agentpulse.dustinnstroud.com?subject=AgentPulse%20paid%20beta%20request&body=I%20want%20to%20join%20the%20AgentPulse%20paid%20beta.%0A%0AName%3A%0ACompany%3A%0AServers%20to%20monitor%3A%0AStack%3A%0ARecurring%20incidents%20I%20want%20fixed%3A%0APreferred%20plan%3A%20Starter%20%2F%20Pro%20Beta%20%2F%20Business%20Beta)

## What to include

- Number of Linux servers you want to monitor.
- What stack you run.
- The incidents that keep repeating.
- Whether you want alert-only, ask-first, or auto-fix behavior.
- Your preferred plan: Starter, Pro Beta, or Business Beta.

## What happens next

We will reply with the recommended first-server setup, the safest initial policies, and the payment/onboarding path for your plan.

The first server starts in alert-only mode. After the first alerts are reviewed, safe actions can move to ask-first or auto-fix. Only promote actions you would be comfortable running over SSH yourself.

## Good first beta fit

- You run 1-10 Linux servers for a real app or client workload.
- You already have alerts, but still manually fix repeat incidents.
- You can name one recurring issue you want reduced: disk pressure, crashed services, runaway processes, or similar.
- You are willing to start with one server and review policies before automation is enabled.
