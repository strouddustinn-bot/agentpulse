## Onboarding Copy

Empty state:

No servers connected yet. Start with one non-critical Linux server in alert-only mode. AgentPulse will identify services, disk pressure risks, and repeat process issues before any remediation policy is enabled.

First server connected:

AgentPulse is learning this server's normal behavior. Keep policies in alert-only mode until you have reviewed the first alerts and promoted the actions you would already run manually.

Policy review:

Only promote actions you would be comfortable running over SSH yourself. Use ask-first for anything that needs judgment.

## Trust Language

AgentPulse is designed to start conservative. Beta onboarding begins in alert-only mode, and risky actions should stay behind ask-first approval until you explicitly promote them. Every remediation policy should be visible, reversible, and scoped to the server where it runs.

Cancellation language:

Cancel during the beta by emailing support. If AgentPulse does not reduce a repeat incident in the first 30 days, the next month is free.

## One-Minute Demo Script

"This is AgentPulse. It is built for founders running a few Linux servers who already have monitoring, but still wake up to run the same fixes.

We start with one server in alert-only mode. AgentPulse watches services, disk pressure, and process behavior. When something repeats, it recommends the first safe fix.

For disk pressure, that might be cleaning known temporary paths. For a crashed worker, it might be restarting the systemd service. For a runaway process, it might ask before killing anything.

The important part is control. Each policy can stay alert-only, move to ask-first, or become auto-fix once you trust it.

The paid beta is $99 per month for up to five servers with onboarding help. The goal is simple: remove one recurring operational headache in the first 30 days."
